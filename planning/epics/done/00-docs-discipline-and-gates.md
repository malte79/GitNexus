Title: Docs Discipline And Gates
Assigned to: Agent 1
Lane: Workflow
Status: done
Objective: Establish a CodeNexus documentation system that documents every new contract and behavior we introduce, enforces markdown and repo-specific doc quality checks, and makes docs validation a required merge gate.
In scope: documentation policy for CodeNexus-owned changes; initial `docs/` information architecture; rules for what must be documented versus explicitly deferred; markdown lint setup; custom doc-contract lint setup; required local merge/prep gates for docs; workflow and epic integration so docs are required for future changes.
Out of scope: documenting the inherited upstream system wholesale; retroactively writing complete docs for existing GitNexus behavior; embeddings docs; Luau/Roblox feature docs before those features exist; CI/provider-specific automation outside the local workflow; general prose cleanup unrelated to the new documentation contract.
Dependencies: `planning/master-intent.md`; `planning/archive/epics-todo.md`; `planning/epics/backlog/01-product-and-runtime-boundary.md`; `AGENTS.md`; `.codex/skills/prep/SKILL.md`; `.codex/skills/merge/SKILL.md`; `gitnexus/package.json`.
Risks: the docs policy could become too broad and accidentally require documenting inherited behavior we do not own; markdown lint alone could create cosmetic churn without protecting contract quality; custom doc checks could be too weak to catch missing docs or too strict to be maintainable; governed docs surfaces could stay ambiguous and accidentally expand to the whole repo; merge gates could be added without clear ownership of which docs must change or whether docs checks are standalone gates versus hidden inside another workflow; workflow docs and actual gates could drift.
Rollback strategy: keep the scope narrow and CodeNexus-owned; if a lint rule or custom doc check causes excessive noise, loosen or remove that check in the same change that updates the docs policy and merge workflow; if the initial docs layout proves wrong, migrate the layout and update all references/gates together rather than allowing parallel undocumented patterns; if a governed-surface rule proves too broad, tighten the governed path set explicitly rather than disabling docs enforcement wholesale.

The policy must also lock that pure refactors with no CodeNexus-owned contract or behavior change do not require docs updates, but must still be allowed only when the docs-contract gate concludes that no governed docs change is required. Done when future implementation epics can be judged for documentation completeness against one explicit policy.
Tests: Validate the policy against representative cases covering a pure refactor with no behavior change, a new CLI contract, a runtime state change, a `.codenexus` layout change, and a new subsystem such as Luau/Rojo support. Validate that inherited upstream areas are explicitly excluded unless touched by CodeNexus-owned behavior changes, that durable docs ownership resolves to `docs/` rather than ad hoc planning locations, and that the policy distinguishes correctly between no-docs-needed refactors and doc-required contract changes.
