Title: Core Surface Reduction
Assigned to: Agent 1
Lane: Architecture
Status: done
Objective: Reduce the repository to the minimum product surfaces needed for the headless CodeNexus core, while preserving the seams required for the upcoming single-repo architecture, CLI reshape, and repo-local MCP HTTP service epics.
In scope: a keep/defer/remove inventory for top-level subprojects, package surfaces, CLI commands, core subsystems, server/MCP layers, workflow/docs surfaces, tests, and dependencies; removal of clearly non-core product surfaces; disabling or quarantining ambiguous surfaces that should not remain first-class; updates to docs and package metadata so the reduced repo accurately reflects the intended product.
Out of scope: implementing repo-local `.codenexus` state ownership; implementing `cn` commands; implementing the repo-local MCP HTTP service; Luau/Roblox/Rojo support; branch-aware indexing; storage-engine changes; full rename sweep from `gitnexus` to `codenexus`; deleting load-bearing seams needed for Epics 03-05.
Dependencies: `planning/master-intent.md`; `planning/epics-todo.md`; `docs/policy.md`; `docs/README.md`; `docs/architecture/product-and-runtime-boundary.md`; `docs/architecture/repo-state-model.md`; `docs/architecture/mcp-http-runtime.md`; `docs/cli/commands.md`; current repo/package surfaces in `gitnexus/package.json`, `gitnexus/src/cli/`, `gitnexus/src/core/`, `gitnexus/src/mcp/`, `gitnexus/src/server/`, top-level subprojects including `gitnexus-web/`, `gitnexus-claude-plugin/`, `gitnexus-cursor-integration/`, `eval/`, `gitnexus-test-setup/`, and repo-level integration scaffolding such as `.claude/`, `.claude-plugin/`, `.github/`, `gitnexus/hooks/`, `gitnexus/skills/`, and `gitnexus/vendor/`.
Risks: deleting surfaces before classifying them could break future implementation seams; removing package dependencies without pruning all references could leave broken builds/tests; deleting user-visible surfaces without updating docs and package metadata could leave the repo lying about what the product is; conflating “non-core” with “unused” could accidentally remove seams still needed for the repo-local MCP/runtime path; failing to distinguish tracked product surfaces from generated or support artifacts could create noisy, low-value deletions while leaving first-class product drift intact; reducing CLI routes without reconciling the command dispatcher and package metadata could leave dead entrypoints or broken help output; keeping too much “for later” would directly violate the reduction goal for this epic.
Rollback strategy: classify before deleting; default every candidate surface to `remove now` unless this epic proves it is required for the headless core or for Epics 03-05; remove whole surfaces in coherent slices; if a cut proves too aggressive, restore it from git history or the upstream clone and reclassify it with explicit rationale rather than preserving speculative keep paths in advance.
